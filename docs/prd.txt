# 제품 요구사항 명세서 (PRD) - 팀 랜덤 선택기 웹앱

## 1. 개요 (Overview)
- **목적**: 팀 내 회식 메뉴 선정, 발표 순서 정하기 등 무작위 선택이 필요한 상황에서 간편하게 사용할 수 있는 웹 애플리케이션 제공.
- **기술 스택**: Next.js (App Router), Tailwind CSS, Cloudflare Pages 접속용 정적 호스팅.
- **주요 특징**: 모던하고 심플한 UI, 동적인 애니메이션 효과, URL 및 텍스트 기반의 손쉬운 데이터 공유.

## 2. 주요 기능 (Features)
- **랜덤 선택 기능**:
  - 항목들 중에서 1개 혹은 그 이상을 무작위로 추첨.
  - 추첨 결과 도출 시 시각적인 강조 및 폭죽(Confetti) 등 축하 애니메이션 제공.
- **데이터 입력 및 공유**:
  - **URL 파라미터 기반 공유**: URL 쿼리 파라미터(`/?names=A,B,C`)를 파싱하여 리스트를 즉시 로드 가능하게 구현 (슬랙 등 메신저에서 간편하게 공유).
  - **직접 텍스트 입력**: 텍스트 에어리어를 통해 줄바꿈이나 콤마로 구분된 텍스트 목록을 붙여넣거나 직접 작성 가능.
- **애니메이션 및 UX**:
  - 부드러운 전환과 선택되는 과정을 보여주는 모던한 애니메이션 구현.
- **반응형 디자인 (Responsive Web)**:
  - 데스크탑, 태블릿, 모바일 기기 모두 최적화된 해상도 지원.

## 3. 비기능적 요구사항 (Non-Functional Requirements)
- **배포 및 호스팅**: Cloudflare Pages를 이용하여 빠르고 안정적으로 서빙하기 위한 정적 사이트(Static Export) 빌드 지원.
- **성능**: 빠른 파싱 및 인터랙티브한 반응 속도 보장.
- **디자인 철학**: 사용자 경험(UX)을 최우선으로, 복잡한 기능 없이 직관적인 싱글 페이지 형태. 세련된 색상 조합과 모던한 폰트 사용.

## 4. 제약 사항 (Constraints)
- 별도의 백엔드 데이터베이스 없이 클라이언트 상태(브라우저 메모리)와 URL 파라미터에만 의존하여 동작.
- 개인 식별이 가능한 민감 정보 수집 및 보관 배제.

## 5. 단계별 개발 계획 (Milestones)
- **Phase 1**: Next.js 템플릿 프로젝트 생성, Tailwind CSS 구성, PRD 산출물 작성 (완료)
- **Phase 2**: 메인 화면 렌더링, 텍스트 입력 및 URL 파라미터 파싱 로직 구현
- **Phase 3**: 랜덤 추첨 로직 및 애니메이션 효과 등 핵심 기능 구현
- **Phase 4**: Cloudflare Pages 배포를 위한 static HTML export 설정 및 배포 테스트
